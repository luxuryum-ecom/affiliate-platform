# OPERATING RULES

- Repository: affiliate-platform
- GitHub is the single source of truth.
- Always inspect existing code before changing anything.
- Never rewrite working features.
- Make the smallest possible change.
- Preserve existing UI/UX unless explicitly requested.
- Do not touch backend or database unless explicitly requested.
- After every task:
  1. Run git status
  2. List modified files
  3. Explain what changed
  4. Do NOT commit
  5. Do NOT push
  6. Wait for user approval

- Default workflow:
  Inspect → Modify → Verify → git status → Report → Wait.
