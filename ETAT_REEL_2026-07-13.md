## DETTE RÉVÉLÉE — GRANTs de base hors migrations → mig 133 (LOCAL, à valider)

> Découverte en marge du correctif (incident local 2026-07-13), **AUCUN impact prod**.

**Symptôme.** En nettoyant un flake local du pré-commit (`lot-e` digest « URI too long »,
dû à ~686 commandes/233 livreurs accumulés en local), un `supabase db reset` a révélé que
la base LOCALE **n'était pas reconstructible depuis les migrations** : **~1231 GRANTs DML**
(`SELECT/INSERT/UPDATE/DELETE` pour `anon`/`authenticated`/`service_role`) sur la quasi-
totalité des tables/vues `public` existaient **en PROD** mais dans **aucun fichier de
migration** (appliqués hors-bande historiquement + pas de `seed.sql`). Après reset, ces
GRANTs manquaient → **34 tests d'intégration en « permission denied »**.

**Diagnostic (LECTURE SEULE prod via `backups/.db_password`, SELECT only).** Diff prod↔local
des `role_table_grants` : prod = 2188 grants, local (post-reset) = 957 → **1231 manquants**,
0 en trop. Erreurs tests = `permission denied for table supplier_products / telegram_inbound
/ profiles / ledger_postings / …`.

**Correctif (LOCAL uniquement).** GRANTs restaurés en local (exacts, générés depuis le diff)
→ base à **2188 grants = prod-identique** → **suite 749/749 verte**. Capturés dans
**`supabase/migrations/133_restore_base_table_grants.sql`** (119 GRANTs groupés exacts + 29
partiels fidèles vues/tables restreintes, **aucun sur-grant**) + `ALTER DEFAULT PRIVILEGES`
(anti-récurrence pour les futures tables).

**Preuve de reproductibilité.** `supabase db reset` (migrations seules, 001→**133**) →
grant count **2188 = prod-identique** → **749/749 verts**. La base est **enfin
reconstructible à l'identique depuis le code**.

**Sécurité.** Ces GRANTs sont *nécessaires-mais-non-suffisants* : la frontière reste la RLS
(deny par défaut, activée partout) + vues redacted SECURITY DEFINER. Un GRANT DML seul
n'expose rien sous RLS deny-par-défaut.

**⚙️ EN ATTENTE ABDOU.** **Mig 133 = LOCAL, NON appliquée en prod, NON mergée** (règle : pas
de merge/push DB sans GO). PROD possède déjà ces GRANTs (hors-bande) → appliquer 133 en prod
= **no-op de mise en conformité du code** (aligne prod sur le code, aucun changement d'accès).
À merger/appliquer sur GO explicite.
